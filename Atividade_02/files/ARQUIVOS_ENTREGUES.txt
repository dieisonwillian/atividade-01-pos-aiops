PostgreSQL:

  DB_USER="postgres"            # Usuário
  DB_HOST="localhost"           # Host
  DB_PORT="5432"                # Porta
  DB_PASSWORD=""                # Senha (use .pgpass!)

AWS:
  AWS_REGION="us-east-1"        # Região AWS
  AWS_ACCESS_KEY_ID="..."       # Key (use ~/.aws/credentials)
  AWS_SECRET_ACCESS_KEY="..."   # Secret (use ~/.aws/credentials)

Backup:
  S3_BUCKET="hvt-ledger-backups"  # Bucket S3
  RETENTION_DAYS="30"           # Dias para manter
  BACKUP_DIR="/var/backups/ledger"
  LOG_FILE="/var/log/ledger-backup.log"

🔒 SEGURANÇA - BOAS PRÁTICAS
═══════════════════════════════════════════════════════════════════════════

AWS:
  ✓ Use IAM user com permissões mínimas (s3:Get, s3:Put, s3:Delete)
  ✓ Armazene credenciais em ~/.aws/credentials (nunca em variáveis)
  ✓ Use MFA se possível
  ✓ Revise permissões regularmente

PostgreSQL:
  ✓ Use .pgpass com chmod 600 (nunca em variáveis)
  ✓ Use user com permissão apenas de backup
  ✓ Não use credenciais root
  ✓ Configure acesso restrito

Geral:
  ✓ Nunca commitar senhas em git
  ✓ Revisar logs regularmente
  ✓ Testar restauração periodicamente
  ✓ Manter cópias de backup em outra região (opcional)

📞 TROUBLESHOOTING RÁPIDO
═══════════════════════════════════════════════════════════════════════════

"pg_dump não encontrado"
  → sudo apt-get install postgresql-client

"Credenciais AWS inválidas"
  → aws configure
  → aws sts get-caller-identity

"Bucket não acessível"
  → aws s3 ls s3://hvt-ledger-backups/
  → Verificar IAM permissions

"Falha ao conectar PostgreSQL"
  → psql -h localhost -U postgres -d postgres -c "SELECT 1;"
  → Verificar .pgpass (chmod 600)

"Sem permissão em /var/backups"
  → sudo chmod 700 /var/backups/ledger/
  → sudo chown postgres:postgres /var/backups/ledger/

Ver todos os logs:
  → tail -100 /var/log/ledger-backup.log
  → grep ERROR /var/log/ledger-backup.log
